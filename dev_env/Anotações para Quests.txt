*****************************
Criar um Cluster GCK
*****************************
> gcloud container clusters create [CLUSTER-NAME] (nome-recurso)

Obter Credenciais
> gcloud container clusters get-credentials [CLUSTER-NAME]

Implantaçào de App no Cluster com yaml
> kubectl create -f caminho/do/arquivo.yaml

Export Serviço do Kubernetes
> kubectl expose deployment nome-server --type=LoadBalancer --port 8080

Verificar serviço instanciado:
> kubectl get service

Encaminhamento de Portas:
> kubectl port-forward container 8080:8080 (Direciona trafego da 8080(Out) para a 8080(in))

Criar regra de Firewall para Liberar acesso as portas
> gcloud compute firewall-rules create allow-nome-container --allow=tcp:8080

Para testar acessos :
> curl -k https://<EXTERNAL_IP>:<Port>

*******************************
Gerenciar Implantações
*******************************

Atualizar numero de Replicas
> kubectl scale deployment container-name --replicas=X (onde x é o numero de réplicas que desejamos manter)

Modificar uma implementação
> kubectl edit deployment container-name

Criar uma implementaçào paralela de nova versào:
> kubectl create -f /camiho/do/novo-aquivo.yaml

Selecione a implementação de produção:
> kubectl apply -f /caminho/do/novo_arquivo.yaml
